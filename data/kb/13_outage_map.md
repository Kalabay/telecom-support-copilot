# Проверка массовых аварий

## Когда применять

**Первый шаг любой диагностики** после уточнения адреса. Если в районе клиента массовая авария — нет смысла гонять его перезагружать роутер.

## Действия оператора

1. В CRM открыть карту аварий / систему инцидентов.
2. Ввести:
   - адрес клиента (улица + номер дома)
   - либо ID узла связи / коммутатора, к которому подключён клиент
3. Проверить статусы:
   - **Активная авария** — есть инцидент, идут работы. Должна быть указана ETA восстановления.
   - **Плановые работы** — см. planned_works
   - **Ничего нет** — продолжить индивидуальную диагностику
4. Если авария:
   - сообщить клиенту факт массовой аварии и ETA
   - **зафиксировать обращение** — клиент в списке пострадавших, при превышении SLA по простою — автоматическая компенсация (compensation)
   - извиниться (apology_script) и предложить альтернативу: мобильный интернет, перенос важных задач
5. Если ETA уже истекло — эскалация на L2 (escalation_l2) с тегом «срыв SLA».

## Рекомендованный ответ

«Сейчас проверю по вашему адресу… Да, действительно, в вашем районе с [время] зафиксирована авария на нашей стороне. Восстановление ожидается к [ETA]. Я добавлю вас в список пострадавших, и за дни простоя автоматически начислится компенсация.»

## Эскалация

- Превышение SLA по аварии — escalation_l2
- Массовое недовольство (флэш-моб обращений) — старший смены / PR-отдел
